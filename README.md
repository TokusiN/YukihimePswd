# ゆき姫のパスワードジェネレータ

## 概要
ゆき姫のたびにっきの生成および解析をするツールです。
ステータスを入力して生成を行う、もしくはたびにっきを入力して解析を行い、
入力欄に反映させる事ができます。

初期値はゲーム開始時およびコンティニュー時の値
最強は通常プレイで入手可能な最大値（ただしステージごとの入手可能なアイテムのチェックはしていません）
カンストは内部値最大の状態を一括で入力できます。
自動入力ではシナリオに関わる部分（ステージ、フラグ、手形などのシナリオアイテム）は変更しません。

## IDについて
パスワードは難読化の為に同一ステータスに対して64種類存在していてランダムに選ばれますが、
どうやら0だけは絶対に選ばれないようになっているようです。

## ROMバージョンについて
1.0、1.1、1.2の3種類が存在し、バージョン間で互換性がありません。1.1と1.2は同じようです。
全バージョンでロジック自体は同じなのですが、
パスワードに格納するデータのアドレスとビット数を管理しているデータをそのまま難読化のためのマスクに使うというコードになっていて、
メモリ配置が変わったためかマスクの値がバージョンによって変わってしまっています。そのため、バージョン間に互換性がありません。
31文字パスワードでは16文字目が異なり、45文字パスワードの後半は大きく異なっています。
解析ボタンは各バージョンで解析を試み、チェックサムが通るバージョンがあればそれを解析結果とし、
どれも通らなければ現在のROMバージョンでの解析結果を反映させるようにしています。

## シナリオフラグの詳細
シナリオの進行度を表すフラグです。
ステージに関わらず8以上だと不正なパスワードになるようです。
ステージ9に限り、6,7だと開始直後にフリーズしてしまいます。それ以外のステージは範囲外の値でも問題ないようです。

ステージ2, 0: 初期状態
ステージ2, 1: 櫓の周りの敵を倒した

ステージ3～7: 恐らく不使用

ステージ8, 0: 初期状態
ステージ8, 1: 琉球王の話を聞いた
ステージ8, 2: シーサー像を壊した

ステージ9, 0: 初期状態
ステージ9, 1: 牢屋の扉を開けてもらった
ステージ9, 2: 殿様の話を聞いた
ステージ9, 3: 牢屋の壁を壊した
ステージ9, 4: 山頂でヤエちゃんの話を聞いた
ステージ9, 5: 大江戸城の門を開いた
ステージ9, 6,7: フリーズ

## 黄金招き猫について
黄金招き猫を取得してからパスワードを取るとどれが取得済みか記録されますが、
パスワードでの再開時にフラグがリセットされるため、意味のないフラグになっているようです。
そのためこのフラグが立っているパスワードは入力直後に聞いても、同一のものを聞くことは出来ません。

## 入力可能な数値について
内部値をそのままパスワードに反映させているようで、フラグ系を除いて一切バリデーションされていません。
入力可能なビット数の範囲で何を入れても通ります。
術回数、所持金、爆弾の個数はBCD（2進化10進数）で管理されているため、10進数にもかかわらず各桁には0～Fの値が入れられます。
その状態でも繰り上がり処理も特に問題なく行えるようです。
鎧、兜の個数は耐久値で管理されていて、個数の表記は耐久値を1個あたりの耐久値で割った値を切り上げて表示しています。
耐久値を使い切った場合、余剰ダメージは他の鎧や本人が受けることはなく、無効化されます。
わらじ、おにぎりは255個まで持てますが、表示が2桁なので10進数の下2桁のみ表示されます。
アイテムを大量に持ったパスワードを生成出来るのは嬉しいのですが、
それぞれ8ビットも確保されているせいでパスワードの文字数が無駄に長くなってしまっています。
最低限必要なビット数に切り詰めていれば9文字短くて済んだようです。

## コンティニューパスワード
解析、生成するメリットが薄いので全パターン載せておきます。
ステージ1～8は内部番号が0～7、ステージ9は9に割り当てられているため、内部番号8のステージが存在しません。
内部番号8の通常パスワードで再開するとステージ9から再開されますが、
コンティニューパスワードで再開するとフリーズします。
プレイヤー0人の通常パスワードで開始すると正常にプレイすることが出来ませんが、
コンティニューパスワードだとゴエモン1人で再開されます。
しかし、何故か参加キャラを含め全ビット0の状態以外のコンティニューパスワードは弾かれてしまいます。
そのため、"るておうあうれ"は7文字なのにゴエモン1人で開始するという特殊なコンティニューパスワードになっています。

# 最初の2文字について
1文字目はパスワードの文字数から4をひいた値、2文字目はマジックナンバーとして7が入っています。
恐らく固定である2文字目を見てIDを判定していると思われます。
1人用の場合、27と7に固定されているため、この64種類の組み合わせしか登場しません。

# 海外版について
生成規則は1.2準拠です。文字コードの対応表はわかったのでコメントアウトして入れてあります。
必要ならば文字コード表をそれに差し替えて生成して下さい。
三角形は^、ハートは♡の文字を割り当ててあります。
解析は終わっているのに英語版対応していないのはこの2文字の解析時の入力が面倒だったからという説が（）
