# ゆき姫のパスワードジェネレータ

## 概要
ゆき姫のたびにっきの生成および解析をするツールです。<br>
ステータスを入力して生成を行う、もしくはたびにっきを入力して解析を行い、<br>
入力欄に反映させる事ができます。<br>

初期値はゲーム開始時およびコンティニュー時の値<br>
最強は通常プレイで入手可能な最大値（ただしステージごとの入手可能なアイテムのチェックはしていません）<br>
カンストは内部値最大の状態を一括で入力できます。<br>
自動入力ではシナリオに関わる部分（ステージ、フラグ、手形などのシナリオアイテム）は変更しません。<br>

## IDについて
パスワードは難読化の為に同一ステータスに対して64種類存在していてランダムに選ばれますが、<br>
どうやら0だけは絶対に選ばれないようになっているようです。<br>

## ROMバージョンについて
1.0、1.1、1.2の3種類が存在し、バージョン間で互換性がありません。1.1と1.2は同じようです。<br>
全バージョンでロジック自体は同じなのですが、<br>
パスワードに格納するデータのアドレスとビット数を管理しているデータをそのまま難読化のためのマスクに使うというコードになっていて、<br>
メモリ配置が変わったためかマスクの値がバージョンによって変わってしまっています。そのため、バージョン間に互換性がありません。<br>
31文字パスワードでは16文字目が異なり、45文字パスワードの後半は大きく異なっています。<br>
解析ボタンは各バージョンで解析を試み、チェックサムが通るバージョンがあればそれを解析結果とし、<br>
どれも通らなければ現在のROMバージョンでの解析結果を反映させるようにしています。<br>

## シナリオフラグの詳細
シナリオの進行度を表すフラグです。<br>
ステージに関わらず8以上だと不正なパスワードになるようです。<br>
ステージ9に限り、6,7だと開始直後にフリーズしてしまいます。それ以外のステージは範囲外の値でも問題ないようです。<br>

## 黄金招き猫について
黄金招き猫を取得してからパスワードを取るとどれが取得済みか記録されますが、<br>
パスワードでの再開時にフラグがリセットされるため、意味のないフラグになっているようです。<br>
そのためこのフラグが立っているパスワードは入力直後に聞いても、同一のものを聞くことは出来ません。<br>

## 入力可能な数値について
内部値をそのままパスワードに反映させているようで、フラグ系を除いて一切バリデーションされていません。<br>
入力可能なビット数の範囲で何を入れても通ります。<br>
術回数、所持金、爆弾の個数はBCD（2進化10進数）で管理されているため、10進数にもかかわらず各桁には0～Fの値が入れられます。<br>
その状態でも繰り上がり処理も特に問題なく行えるようです。<br>
鎧、兜の個数は耐久値で管理されていて、個数の表記は耐久値を1個あたりの耐久値で割った値を切り上げて表示しています。<br>
耐久値を使い切った場合、余剰ダメージは他の鎧や本人が受けることはなく、無効化されます。<br>
わらじ、おにぎりは255個まで持てますが、表示が2桁なので10進数の下2桁のみ表示されます。<br>
アイテムを大量に持ったパスワードを生成出来るのは嬉しいのですが、<br>
それぞれ8ビットも確保されているせいでパスワードの文字数が無駄に長くなってしまっています。<br>
最低限必要なビット数に切り詰めていれば9文字短くて済んだようです。<br>

## コンティニューパスワード
解析、生成するメリットが薄いので全パターン載せておきます。<br>
ステージ1～8は内部番号が0～7、ステージ9は9に割り当てられているため、内部番号8のステージが存在しません。<br>
内部番号8の通常パスワードで再開するとステージ9から再開されますが、<br>
コンティニューパスワードで再開するとフリーズします。<br>
プレイヤー0人の通常パスワードで開始すると正常にプレイすることが出来ませんが、<br>
コンティニューパスワードだとゴエモン1人で再開されます。<br>
しかし、何故か参加キャラを含め全ビット0の状態以外のコンティニューパスワードは弾かれてしまいます。<br>
そのため、"るておうあうれ"は7文字なのにゴエモン1人で開始するという特殊なコンティニューパスワードになっています。<br>

# 最初の2文字について

恐らく固定である2文字目を見てIDを判定していると思われます。<br>
1人用の場合、27と7に固定されているため、この64種類の組み合わせしか登場しません。<br>

# 海外版について
生成規則は1.2準拠です。文字コードの対応表はわかったのでコメントアウトして入れてあります。<br>
必要ならば文字コード表をそれに差し替えて生成して下さい。<br>
三角形は^、ハートは♡の文字を割り当ててあります。<br>
解析は終わっているのに英語版対応していないのはこの2文字の解析時の入力が面倒だったからという説が（）<br>
